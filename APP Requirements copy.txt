# Wheel Trader Strategy With Kite - Web App Requirements

## Goal
Build a local web app for managing a Wheel Trader / conservative income options workflow using Zerodha Kite. The app should help generate, review, dry-run, and place option orders safely.

## Core Tabs

### 1. Place Order
- Read order input from `src/script/kite_orders.csv`.
- Show the CSV contents in an editable text box by default.
- Allow user to paste or upload CSV.
- On load/preview, update `kite_orders.csv` from the text box.
- Before updating, archive the previous file as a record file:
  `kite_orders_last_input_order_YYYY-MM-DD_HHMMSS.csv`
- Parse and display CSV orders in a table.
- Allow selecting all or selected trades.
- Support dry run.
- Support live execution only when `KITE_CONFIRM_LIVE_ORDER=YES`.
- Use Kite order placement logic from existing scripts.
- Show Kite/script console output inside the web app.

### 2. Current Position BUY Options
- Fetch current open Kite positions.
- Generate BUY orders for existing positions using logic similar to `kite_buy_positions.py`.
- Default behavior should cover short positions.
- Options should include:
  - Discount percent
  - Exchange
  - Product filter
  - Symbol filter
  - Variety
  - Validity
  - Tag
  - Tick size
  - Max orders
  - Include long positions
  - Profit-only positions
  - Autoslice
  - Place new order instead of modifying similar open order
- Show generated BUY orders in a selectable table.
- Allow dry run or live placement.
- Show P&L as whole number only, without decimal part.
- Keep detailed Position BUY options at the bottom of the page.

### 3. GPT CSV Generator
- Allow generating Kite CSV orders using OpenAI API.
- Inputs:
  - `OPENAI_API_KEY`
  - Model
  - Prompt
- Output must be valid CSV with header:
  `exchange,tradingsymbol,quantity,transaction_type,product,order_type,price,validity`
- Allow paste fallback from ChatGPT share/conversation.
- Extract CSV from pasted GPT output.
- Save generated CSV to `kite_orders.csv`.
- Save and preview generated orders.
- Existing ChatGPT share URL can be shown, but app should not depend on ChatGPT login pages.

### 4. Kite Setup
- Move Kite credentials into separate setup tab.
- Fields:
  - `KITE_API_KEY`
  - `KITE_API_SECRET`
  - `KITE_ACCESS_TOKEN`
  - `KITE_CONFIRM_LIVE_ORDER`
- Add option to show/hide credential values.
- Other tabs should use saved setup values through hidden fields/session environment.

## Top Dashboard

### Header
- Compact header, reduced height.
- Show app name: `Kite Trader`.
- Center text:
  `ॐ | Jai Sri Ram | Jai Laxmi Mata`
- Show Naval quote:
  `"Trade money for time, not time for money. You're going to run out of time first." - Naval`

### Safety Rules
Show compact top cards:
- `SELL only when you hold the stock`
- `SELL PUT only when you have cash`

### MMI
- Show Market Mood Index value and mood.
- Fetch from Tickertape when possible.
- Provide link to open MMI page.

### Expiry Information
- Show expiry month, expiry date, and trading days remaining in one line:
  `MAY 2026 | Expiry: Tuesday, 26 May 2026 | 8 trading days remaining`
- Calculate expiry from option symbols.
- Use NSE monthly expiry rule.
- Show warning only when 5 trading days or fewer remain:
  `DO NOT take new position 5 trading DAYS near to expiry`
- Warning should flash when visible.

### Watchlist Quotes
- Top ticker/quote cards should not come from CSV.
- Use fixed watchlist:
  - `NSE:BAJFINANCE`
  - `NSE:TATACONSUM`
  - `NSE:PGEL`
  - `NSE:TITAN`
  - `NSE:ETERNAL`
  - `NSE:UNITDSPR`
  - `NSE:HAVELLS`
  - `NSE:NAUKRI`
  - `NSE:PFC`
  - `NSE:CAMS`
  - `NSE:CDSL`
  - `NSE:MAZDOCK`
  - `NSE:NUVAMA`
  - `NSE:NTPC`
  - `NSE:WAAREEENER`
- Fetch LTP and day % change from Kite.
- Display compact cards.
- If day change is more than `+2%`, card background should be light green.
- If day change is less than `-2%`, card background should be light red.

## UI Requirements
- Main action tabs should be larger:
  - Place Order
  - Current Position BUY Options
- Utility tabs should be smaller:
  - GPT CSV Generator
  - Kite Setup
- Use lighter colors in top cards.
- Keep dashboard compact to save vertical space.
- Avoid boring/plain UI; make it feel like a trading dashboard.
- Keep forms practical and dense for repeated trading workflow.

## Safety Requirements
- Dry run should be default.
- Live order placement must require `KITE_CONFIRM_LIVE_ORDER=YES`.
- Never place live orders unless user explicitly disables dry run.
- Show clear execution results and Kite console messages.
- Preserve previous CSV input files before overwriting.
